Connect GitHub to Tembo to open pull requests, review code, and respond to issues and comments.

## Installation

<Steps>
  <Step title="Install">
    Go to [Integrations](https://app.tembo.io/integrations) and click **Install** next to GitHub.
  </Step>
  <Step title="Authorize">
    Choose **All repositories** or **Selected repositories** and authorize.
  </Step>
  <Step title="Activate">
    Select which repositories Tembo should work with under "Active Repositories".
  </Step>
</Steps>

To disconnect, go to Integrations and click **Disconnect**.

## Using @tembo in GitHub

### PR comments

Comment `@tembo` on any pull request to request changes. Tembo reads the feedback, updates the code, and pushes new commits.

```
@tembo Please add error handling to this function
```

Works on PRs created by Tembo and PRs created by your team.

### Issues

Comment `@tembo` on any issue to start implementation. Tembo reads the issue title and description for context, then opens a PR.

```
@tembo Implement the API endpoint described in this issue
```

### Failed workflows

Tembo monitors GitHub Actions and detects failures. When a workflow fails (broken tests, lint errors), Tembo analyzes the failure and opens a fix PR.

## Webhooks

Tembo listens to the following GitHub events for [automation triggers](/features/automations):

- `pull_request.opened`, `pull_request.closed`, `pull_request.reopened`
- `pull_request.ready_for_review`, `pull_request.edited`
- `pull_request_review.submitted`
- `issue_comment.created`

Need an event not listed here? Contact support@tembo.io.

## Signed commits

All commits in Tembo PRs are automatically signed and verified with a "Verified" badge in GitHub.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Repositories not appearing">
    Wait 2-5 minutes for initial sync. If they still don't show, click the sync button on the GitHub integration card. Check that you granted Tembo access to the repositories during installation and that you have permission to install GitHub Apps for your organization.
  </Accordion>

  <Accordion title="@tembo not responding">
    Make sure you're using `@tembo` (lowercase). Verify the repository is activated in Integrations and that `issue_comment.created` webhook is enabled.
  </Accordion>

  <Accordion title="PRs not being created">
    Check that the repository is activated, Tembo has permission to create branches and open PRs, and there isn't already an open PR for the same issue.
  </Accordion>
</AccordionGroup>
