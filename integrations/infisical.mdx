## Features

- Load secrets at runtime instead of storing sensitive values in the repository
- Support shared project secrets with developer-specific personal overrides
- Use one secret source across API, web, worker, and cron commands
- Keep local and CI-style runs consistent with the same Infisical environment

## Installation

<Steps>
  <Step title="Open Sandbox settings">
    In Tembo, go to **Settings** -> **Sandbox**.
  </Step>
  <Step title="Add environment variable">
    Add `INFISICAL_TOKEN` to your sandbox environment variables and save.
  </Step>
  <Step title="Start a new session">
    Start a new Tembo session so the updated sandbox environment is applied.
  </Step>
</Steps>

## Usage

The Infisical CLI is already installed in Tembo sandboxes.

Wrap app commands with `infisical run --`:

```bash
infisical run -- bun run --watch index.ts
infisical run -- next dev --turbopack --port 3000
```

Use Infisical personal overrides for developer-specific values (for example local `DATABASE_URL` or personal GitHub App credentials) while keeping shared team defaults in the project.

## Advanced

<AccordionGroup>
  <Accordion title="How Tembo picks project and environment">
    For `infisical run`, Tembo can use default project, environment, path, and domain settings when they are provided in the runtime environment. If you pass explicit CLI flags, your explicit values take precedence.
  </Accordion>

  <Accordion title="Universal Auth support">
    Tembo supports token-based access and Universal Auth for non-interactive environments. If your setup uses Universal Auth, provide the required client credentials in the runtime environment.
  </Accordion>

  <Accordion title="Secrets are not showing up">
    Confirm the command is started with `infisical run -- ...`, verify you are targeting the expected project/environment/path, and check that the required keys exist in Infisical.
  </Accordion>
</AccordionGroup>
