## Features

- Monitor slow queries and suggest optimizations
- Detect missing and unused indexes
- Open PRs with migration scripts
- MCP server for read-only database access during sessions

## Installation

<Steps>
  <Step title="Create user (optional)">
    Create a dedicated read-only database user for Tembo:
    ```sql
    CREATE USER tembo_monitor WITH PASSWORD 'your-secure-password';
    GRANT CONNECT ON DATABASE your_database TO tembo_monitor;
    GRANT USAGE ON SCHEMA public TO tembo_monitor;
    GRANT SELECT ON ALL TABLES IN SCHEMA public TO tembo_monitor;
    ```
  </Step>
  <Step title="Connect">
    Go to [Integrations](https://app.tembo.io/integrations) and click **Connect** next to Postgres. Enter your connection string.
  </Step>
  <Step title="Map repository">
    Map your database to the repository where migrations are stored so Tembo can open optimization PRs.
  </Step>
</Steps>

## Usage

Tembo monitors your database using `pg_stat_statements`. When it finds slow queries or missing indexes, it opens a PR with migration scripts and expected improvement metrics.

Requires `pg_stat_statements` extension: `CREATE EXTENSION IF NOT EXISTS pg_stat_statements;`

## Advanced

<AccordionGroup>
  <Accordion title="MCP server">
    The Postgres MCP runs in read-only mode. It may access your database during sessions when querying would help complete the work.
  </Accordion>

  <Accordion title="Best practices">
    Use read-only access. Connect production databases for accurate insights. Always review migration PRs before merging, especially for large tables.
  </Accordion>
</AccordionGroup>
