Artifacts are session outputs that are better reviewed outside the chat transcript. Agents create artifacts for rich walkthroughs, diagrams, dashboards, reports, structured exports, screenshots, PDFs, and other files.

## Artifact types

| Type | Use for |
| ---- | ------- |
| **HTML** | Interactive reports, diagrams, dashboards, PR walkthroughs, timelines, and visual summaries |
| **Markdown** | Prose reports, audits, runbooks, checklists, and migration summaries |
| **CSV** | Tables, inventories, exports, and comparison matrices |
| **JSON** | Structured data, API outputs, manifests, and machine-readable exports |
| **Asset** | Uploaded screenshots, videos, PDFs, archives, and other files |

Inline HTML, Markdown, CSV, and JSON artifacts are limited to 5 MiB after UTF-8 encoding. Larger outputs are uploaded as asset artifacts.

## View artifacts

Open a session and select **Artifacts** in the sidebar. Tembo groups artifacts into sections for sources, files, plans, and generated outputs.

HTML artifacts open in a preview with actions to copy a link, download the file, or open the artifact full screen. Files and plans open in a preview dialog on desktop and a bottom sheet on mobile.

## When agents create artifacts

Agents create artifacts when a self-contained output is easier to scan, share, or revisit than a long chat response. Good examples include:

- PR walkthroughs
- Bug explainers
- Architecture maps
- Security findings
- License audits
- Incident timelines
- Cloud cost maps
- UX variations
- Release checklists
- Rollout plans

Artifacts stay attached to the session and use the same organization and session access controls as the rest of the work.
