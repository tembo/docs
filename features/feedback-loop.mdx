---
title: "Feedback Loop"
description: "Iterate on Tembo's work by leaving feedback, just like you would with any teammate."
---

Tembo opens a PR. You review it. If something needs to change, mention `@tembo` in a comment and describe what you want. Tembo reads your feedback, updates the code, and pushes new commits. Keep iterating until it's right.

Works on GitHub, GitLab, and Bitbucket (use `/tembo` instead of `@tembo` on Bitbucket).

## How to give feedback

**PR review comments** (recommended): Use your Git provider's review feature. Comment on specific lines and tag `@tembo`. Tembo knows exactly which code you're referring to.

**Regular comments**: Leave a comment on the PR and mention `@tembo`.

Only comments that mention `@tembo` are picked up. This prevents Tembo from responding to internal team discussions.

## Examples

```
@tembo Add retry logic with exponential backoff and a unit test
```

```
@tembo This function should handle the case where the user doesn't exist
```

```
@tembo Refactor this to use the existing auth middleware instead
```

## What happens

1. Tembo reacts with 👀 (seen) then 🚀 (working)
2. New commits appear on the PR within a few minutes
3. If the task started from Slack, Tembo posts a summary back to the thread

Be specific. `@tembo add error handling for timeouts` works better than `@tembo make it better`.
