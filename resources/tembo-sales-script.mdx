## Elevator Pitch (30 seconds)

Tembo is your unified interface for background coding agents. ## The Problem We Solve

Engineering teams are drowning in repetitive work:

- **Production bugs** pile up in error tracking tools like Sentry
- **Technical debt** accumulates as teams focus on features
- **Documentation** falls behind the codebase
- **Code reviews** and PR iterations create bottlenecks
- **Project management tools** (Linear, Jira) fill up with small tasks that never get prioritized
- **CI/CD failures** require manual investigation and fixes

The result? Engineers spend 30-40% of their time on toil instead of building what matters.

## What is Tembo?

Tembo is a **background coding agent platform** that autonomously handles engineering tasks across your entire development workflow.

### Core Capabilities

1. **Automated Bug Fixes**
   - Monitors Sentry, AWS CloudWatch, and other error tracking tools
   - Automatically generates PRs to fix production errors
   - Reduces MTTR (Mean Time To Resolution) from hours to minutes

2. **Issue Implementation**
   - Assign issues from Slack, Linear, Jira, or GitHub
   - Tembo writes code to solve bugs, build features, or refactor
   - Works across multiple repositories simultaneously

3. **Technical Debt Reduction**
   - Periodic codebase analysis for security vulnerabilities
   - Identifies refactoring opportunities and performance issues
   - Submits PRs for quick wins automatically

4. **Documentation Automation**
   - Syncs monorepo changes to docs repositories
   - Keeps feature documentation current
   - Generates changelogs from commit history

5. **CI/CD Monitoring**
   - Detects failed workflows and broken tests
   - Investigates root causes and pushes fixes
   - Reduces "red pipeline" incidents

## How It Works

### 1. Detection
Issues are identified through:
- **Integration signals**: Sentry errors, database metrics, failed CI/CD
- **Manual assignment**: Linear/Jira tickets, GitHub issues
- **Scheduled scans**: TODO comments, security vulnerabilities, technical debt
- **Direct creation**: Tembo dashboard or Slack

### 2. Execution
- Tembo analyzes the codebase using your configured coding agent
- Generates solution in an isolated sandbox environment (Docker + VM)
- Automatically detects and uses Nix dev shells if present
- Full access to MCP servers for extended capabilities

### 3. Pull Request
- Changes submitted as PR with detailed context
- Links to original issue and explains approach
- Includes test results and validation steps

### 4. Feedback Loop
- Mention `@tembo` in PR comments to request changes
- Tembo reads all feedback, updates code, and pushes new commits
- Integrates with AI code review tools (Graphite, CodeRabbit, Cursor Bug Bot)

## Common Objections & Responses

### "We already have GitHub Copilot"
Copilot is excellent for code suggestions, but Tembo handles **complete tasks autonomously**. Copilot requires a developer to drive; Tembo works in the background. They complement each other ‚Äî Copilot for writing code, Tembo for maintaining and fixing it.

### "Our code is too complex for AI"
That's exactly why Tembo supports multiple coding agents and models. Start with simpler tasks (bug fixes, docs updates) and gradually expand. Our sandbox environment ensures safe execution, and you review all PRs before merge.

### "Security concerns about AI accessing our codebase"
- SOC 2 Type II certified
- All data encrypted in transit and at rest
- Self-hosted deployment available for enterprise
- Granular access controls per repository
- Audit logs for all actions
- Signed commits for full transparency

### "What if Tembo makes a mistake?"
- All changes come as PRs, you review before merge
- Sandbox environment prevents production impact
- Failed tasks are logged with full context
- @tembo feedback loop allows quick iterations
- Integration with existing code review tools

### "How is this different from AutoGPT or LangChain agents?"
Tembo is purpose-built for engineering workflows, not general AI automation:
- Pre-built integrations with dev tools
- Git operations are reliable and production-tested
- Multi-repository coordination
- Team collaboration features (task assignment, notifications)
- Enterprise security and compliance
