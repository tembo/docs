Agent pre-run scripts let you run a Bash script in the sandbox before an agent starts. Use them when an agent needs repository setup, generated files, dependency checks, or environment preparation that should happen after Tembo prepares the workspace and before the agent begins work.

## Prerequisites

- Agent pre-run scripts must be enabled for your workspace.
- You need permission to edit the agent.
- The script must be safe to run in the agent sandbox.

## Configure a pre-run script

1. Open the agent you want to configure.
2. Add your Bash script in the pre-run script field.
3. Save the agent.
4. Trigger the agent.

The script runs in `/workspace`, after Tembo sets up the repository and before the agent harness starts.

## Example

```bash
#!/usr/bin/env bash
set -euo pipefail

npm ci
npm run generate
```

## How output affects the agent

When the script succeeds, Tembo passes its output to the agent as pre-run context. The agent can use that output to understand what setup completed, which files changed, or what checks already ran.

When the script exits with a non-zero status, Tembo stops the run before the agent starts. Use this behavior for required setup or validation that should block the agent from working with an incomplete environment.

## Trigger payloads

For triggered agent runs, Tembo exposes the trigger payload to the script through the `TRIGGER_PAYLOAD` environment variable.

```bash
#!/usr/bin/env bash
set -euo pipefail

echo "$TRIGGER_PAYLOAD" | jq .
```

Use this when the setup step depends on event data, such as a GitHub issue, Slack message, webhook body, or other trigger source.

## Workspace resumes

Tembo skips the pre-run script when resuming an existing workspace. If a job is retried, the script runs again before the agent starts.
